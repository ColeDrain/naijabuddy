# NaijaBuddy — Numbers Integrity

Every quantitative claim in `solution_paper.md` Section 4, mapped to the exact
command that regenerates it. All evaluation is leakage-free leave-one-out; the
LLM artifact cache (`eval_artifacts/llm_cache.jsonl`) makes re-runs deterministic
and free of new GPU cost. Domains are always listed Yelp / Goodreads / Amazon.

The headline figures are from the **multi-seed n = 2,000-user evaluation** —
three independent leave-one-out splits (seeds 42 / 1 / 7), n = 2,000 held-out
interactions per seed, synth personas, served by vLLM-batched Qwen2.5-3B-Instruct
(fp16). All cross-domain numbers below are reported as **mean ± sample std**
across the three seeds. The canonical run is:

```
# one per seed, three parallel Modal A10G containers
modal run modal_vllm_eval.py --sample 2000 --persona-mode synth \
    --seed {42,1,7} --cold-start --cold-sample 2000 --bertscore
```

Held-out set sizes per seed: 2,000 / 2,000 / 1,999.

## Discarded / superseded figures

- An earlier draft scored the LLM blend on an **n = 10** sample and reported a
  Goodreads V2 RMSE of **0.60**. The full-set evaluation corrects this to
  **~0.88** — the 0.60 was sampling noise and is **not used anywhere in the
  paper**. See `EXPERIMENTS.md` #1.
- The earlier **n = 350** multi-seed run (V2 0.958 / 0.937 / 0.784) is
  superseded by the n = 2,000 multi-seed run above. Every *qualitative* finding
  of that run — regime switch, V1 dominates warm, LLM weight ≈ 0, two populations,
  CF ≥ hybrid — reproduced at n = 2,000 across three seeds; magnitudes shifted
  but cross-seed std is now an actual error bar, not extrapolated from sample
  size. See `EXPERIMENTS.md` #6 vs #10 vs #16.
- The **single-seed n = 2,000 template+llama-cpp-Q4 run** (`EXPERIMENTS.md` #10)
  is superseded by the **3-seed n = 2,000 synth+vLLM-fp16 run** (#16). The two
  engines agree to within the std (V2 shift ≤ 0.018 in every domain) — engine
  precision change is dominated by sampling noise. The earlier single-seed
  numbers are retained as the §4.6 / §4.7 / §4.2-V3 / §4.3-BERTScore-F1
  reference values where the new multi-seed run did not (or could not) replace
  them.
- V2 / V3 blend weights are the test-minimising point of an α-sweep (a
  descriptive minimum, not a validation-tuned hyperparameter). This makes the
  "LLM barely helps warm" verdict conservative, not optimistic, and is disclosed
  in §4.2. See `validation_audit.md`.

## Reproduction notes

- Results are written to `evaluation_results.json` and `evaluation_results.md`.
- The n = 2,000 dense datasets that back the figures above are **regenerated
  deterministically by `python local_data_prep.py`** (it streams the three HF
  source datasets and rebuilds `data/*_dense.csv`). They total ~592 MB and,
  pending a hosting decision, are **not committed** — the committed
  `data/*_dense.csv` is still the earlier n = 350 build. Synthesised personas
  (`data/synthesized_personas.json`) and item embeddings
  (`data/item_embeddings.npz`) are committed. Run `local_data_prep.py` once and
  every command above then reproduces fully offline.
- The complete per-run ledger — config, headline numbers, verdict — is
  `EXPERIMENTS.md`.
